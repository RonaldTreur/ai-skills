# Discipline Map

Use this map to review external sources against the real project-development routine.

## Core Project Disciplines

- Research and discovery
- Product thinking and design
- UI design
- Architecture
- Implementation planning
- Implementation
- Debugging
- Testing and QA
- Code review
- Documentation and handoff
- Learning capture and skill promotion
- Agent orchestration and delegation

## Meta-Behavior Disciplines

- When to slow down
- When to ask Ronald
- When to act without asking
- When to delegate
- When to stop digging and answer
- How to preserve attribution and provenance
- How to revisit external sources after they change

## First Recommended Vertical Slice

Start with debugging.

Reason: debugging has clear failure modes, strong candidate source material, and obvious ROI. A good debugging discipline can prevent speculative fixes, missing repros, weak verification, and forgotten cleanup.
